# 🚗 미션 - 자동차 경주 게임

## 🚩 구현 기능 목록

- **`기능1`** [입력처리]n대의 자동차를 입력받는다.

    - `1-a` [예외처리]`,`를 기준으로 구분한다.
      
       - [예외처리]`,` 구분자가 맨 앞이나 맨 뒤에 나올 순 없다.
       - [예외처리]`,`만 입력되는 경우 예외처리한다.
       - [예외처리]`,` 뒤에는 자동차 이름이 무조건 있어야한다.    
        - [예외처리]자동차가 1대일 경우 ','를 기준으로 자르지 않는다.

    - `1-b` [예외처리]5자 이하여야한다.
    - `1-c` [예외처리]문자만 입력받는다.
    - `1-d` [예외처리]중복되는 이름을 금지한다.
    
- **`기능2`** [입력처리]시도할 횟수를 입력받는다.

    - `2-a` [예외처리]입력은 문자로 한다. (예외처리를 위해서)
    - `2-b` [예외처리]숫자가 아닌 값을 입력한 경우 예외처리해준다.
    - `2-c` [예외처리]공백이 입력되는 경우 예외처리해준다.
    - `2-d` [예외처리]숫자만 입력받는다.
    - `2-e` [예외처리]음수일 경우 예외처리해준다.
    - `2-f` [예외처리]0이 입력되는 경우 예외처리해준다.
    
- **`기능3`** 입력받은 횟수만큼 라운드를 진행한다.

    - `3-a` 각 자동차마다 전진/멈춤을 시행한다.
        
        - 각각의 자동차마다 random값을 부여한다.
        - random 값은 0과 9 사이의 값으로 제한한다.    
        - 4 이상이면 전진, 3 이하이면 멈춘다.
        
    - `3-b` 매 회마다 모든 자동차가 전진/멈춤을 시행하고 난 뒤, 현재 자동차의 거리를 출력한다.
    - `3-c` 매 회마다 가장 멀리 이동한 거리를 확인한다.
    
- **`기능4`** 최종 우승자를 출력한다.

    - `4-a` 매 회별로 업데이트된 최대이동거리를 갖는 자동차의 이름을 출력한다.

## 📌 프로그램 구성 요소

- 📁 racingcar

    - 📁 controller
    
        - 📃 RaceController : 경주의 작동 순서에 맞게 객체에 데이터 요청, 출력하는 기능을 제어합니다.
    
    - 📁 domain
    
        - 📃 Car : 자동차에 관한 정보를 가진 클래스입니다.
        - 📃 Cars : 경주에 참여하는 자동차들의 데이터에 대한 접근을 제어합니다. 
    
    - 📁 view
        
        - 📃 View : 기능에 따라 input/outputView의 작동 순서를 제어합니다.
        - 📃 InputView : 사용자의 입력에 대한 기능을 담당합니다.
        - 📃 OutputView : 사용자의 출력에 대한 기능을 담당합니다.
     
    - 📃 Application : 어플리케이션 실행 기능을 담당합니다. 
    
- 📁 utils
 
    - 📁 validator
    
        - 📃 InputValidator : 입력 시 발생하는 예외 처리를 담당합니다.
        - 📃 NameValidator : 이름 입력 시 발생하는 예외 처리를 담당합니다.
        - 📃 TimeValidator : 횟수 입력 시 발생하는 예외 처리를 담당합니다.
    
    - 📃 RandomGenerator : 랜덤 숫자의 범위를 저장, 랜덤 숫자 모듈 호출을 담당합니다.
    - 📃 RandomUtils : 랜덤 숫자를 생성하는 기능을 담당합니다.
